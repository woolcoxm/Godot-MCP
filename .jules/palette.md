## 2024-05-03 - Status command UX in Godot Editor Plugin
**Learning:** For Godot editor plugins, user-triggered commands (like a "Show Status" menu item) should utilize a UI dialog (such as `AcceptDialog`) rather than exclusively logging to `print()`. Console output might be hidden from the user, leading to a confusing experience where actions appear to have done nothing. Furthermore, dynamically instantiated UI dialogs in Godot 4 do not auto-delete on closure; they must be manually freed (`queue_free()`) upon `confirmed` or `canceled` signals to prevent memory leaks. They also need to be parented to the editor's base control (`get_editor_interface().get_base_control()`) to ensure proper styling.
**Action:** Always wrap user-triggered status readouts in an `AcceptDialog`, attach the memory management signals, and parent them to the editor base control.
