## 2025-02-14 - Prevent Command/Flag Injection in External Process Execution
**Vulnerability:** User-provided paths (like `projectPath`) and untrusted arguments in tools like `godot_launch_editor` and `godot_run_project` were being passed directly to `child_process.spawn()` without validation. This could allow flag injection (e.g., `-s`, `--script`, `--headless`) by manipulating the `projectPath` to start with a hyphen, or executable manipulation via `editorPath`.
**Learning:** Even when using `spawn` with an array of arguments (which prevents shell injection), flag injection is still a critical risk if user inputs can be interpreted as command-line flags by the target executable (Godot).
**Prevention:** Created `src/utils/security.ts` to strictly validate executable names using `isValidExecutable`, block paths starting with a hyphen using `isPathSafe`, and sanitize arrays of user arguments with `sanitizeUserArguments` to explicitly reject prohibited flags like `--script`, `-s`, `--export`, and `--headless`. These checks must be applied before invoking `spawn` with any untrusted inputs.
