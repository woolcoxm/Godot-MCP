## 2024-05-14 - Prevent Command and Argument Injection in Editor Tools
**Vulnerability:** The `launch-editor.ts` and `run-project.ts` tools passed unsanitized user inputs (`projectPath`, `editorPath`, `args`) directly to `child_process.spawn`. This allowed for arbitrary executable execution (if `editorPath` was manipulated) and flag injection (if `projectPath` started with `-` or if `args` contained execution flags like `--script`).
**Learning:** Even when inputs are typed as strings via Zod schemas, they can still introduce command injection vulnerabilities when passed to OS execution functions. Implicit trust of paths or arguments allows attackers to alter the process's execution context.
**Prevention:** Always validate executables against an allowlist pattern (e.g., must start with `godot`), ensure paths don't start with flag indicators (like `-`), and filter out dangerous arguments (like `--script`, `-s`, `--export`) using centralized security utilities before invoking external processes.
