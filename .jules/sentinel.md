## 2026-05-08 - Command Injection via Godot Process Spawning
**Vulnerability:** The Godot MCP server uses `child_process.spawn` to launch Godot executables. Untrusted user inputs, specifically `projectPath`, `editorPath`, `exportPath`, and arbitrary `args`, were passed directly to `spawn` without validation, allowing for arbitrary command execution by injecting Godot-specific flags like `--script`, `--export`, `--headless`.
**Learning:** `spawn` prevents traditional shell injection (like `&&` or `;`), but is still vulnerable to argument injection when the target executable (like `godot`) has dangerous flags that can execute arbitrary code (e.g. `--script`).
**Prevention:** Implement strict input validation for all values passed to `spawn`. Ensure executable names are allowed (e.g., must start with `godot`), validate that paths do not start with hyphens (to prevent them from being interpreted as flags), and block known dangerous arguments such as `--script` and `--export` for user-provided argument arrays.
